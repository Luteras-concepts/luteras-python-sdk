# Contributing to LUTERAS Python SDK

Thank you for your interest in contributing to the LUTERAS Python SDK.

We welcome bug reports, feature requests, documentation improvements, and code contributions.

---

## Development Setup

Clone the repository:

```bash
git clone https://github.com/Luteras-concepts/luteras-python-sdk.git
cd luteras-python-sdk
```

Create a virtual environment:

```bash
python -m venv venv
source venv/bin/activate
```

Install dependencies:

```bash
pip install -e .
```

---

## Running Tests

When unit tests are available:

```bash
pytest
```

---

## Coding Guidelines

- Follow PEP 8.
- Write clear, readable code.
- Keep functions focused.
- Add docstrings for public methods.
- Preserve backward compatibility whenever possible.

---

## Reporting Bugs

Please use the GitHub Bug Report template and include:

- Python version
- SDK version
- Operating system
- Steps to reproduce
- Expected behavior
- Actual behavior

---

## Requesting Features

Use the GitHub Feature Request template.

Please explain:

- What problem you're solving
- Why the feature is useful
- Example usage

---

## Pull Requests

Before submitting a pull request:

- Ensure the SDK builds successfully.
- Update documentation if necessary.
- Add tests for new functionality when applicable.
- Keep pull requests focused on one change.

---
